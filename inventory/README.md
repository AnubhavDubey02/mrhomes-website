# Mr Homes Realtors — Inventory Workspace

Private operational workspace for ingesting, normalising, and publishing property inventory.
**This folder is not served by the website and does not affect the Next.js build.**

---

## Folder Reference

```
inventory/
├── raw/                   ← incoming raw data, one subfolder per source
│   ├── s2l/               ← Sell2Live feed (screenshots / exports)
│   ├── fnf/               ← Fully Furnished & Fitted channel
│   ├── dore/              ← Dore broker network
│   └── local-brokers/     ← WhatsApp forwards from individual brokers
│
├── processed/             ← OCR output and parsed JSON records (auto-generated)
│
├── output/
│   ├── master/            ← Final approved inventory ready for website import
│   ├── review/            ← Uncertain / low-confidence records needing human review
│   ├── logs/              ← Extraction run logs (timestamps, error counts, field coverage)
│   └── exports/           ← CSV / XLSX exports for sharing or CRM upload
│
├── photos/
│   ├── temp/              ← Raw property photos before sorting
│   └── listings/          ← Photos organised by listing ID (e.g. listings/MRH-0042/)
│
├── scripts/               ← OCR, extraction, and normalisation scripts
│
└── config/                ← All pipeline configuration and mapping rules
    ├── normalization-rules.json
    ├── floor-mapping.json
    ├── source-priority.json
    └── sector-mapping.json
```

---

## Folder Details

### `raw/`
Drop incoming inventory data here, sorted by source. Accepted formats: PNG screenshots, PDF exports, XLSX / CSV files.

| Subfolder | Source | Trust Level |
|-----------|--------|-------------|
| `s2l/` | Sell2Live | High |
| `fnf/` | Fully Furnished & Fitted channel | High |
| `dore/` | Dore broker network | Medium |
| `local-brokers/` | WhatsApp forwards | Low — manual review required |

Files in `raw/` are **never modified**. They are the source of truth for audit purposes.

---

### `processed/`
Auto-generated by OCR and extraction scripts. Contains one JSON file per raw source file, named identically (e.g. `raw/s2l/batch-01.png` → `processed/s2l-batch-01.json`).

Do not edit files here manually — they are overwritten on re-run.

---

### `output/master/`
Final, deduplicated, normalised inventory. These records are safe to import into the website's property data layer.

Canonical record schema:
```json
{
  "id": "MRH-0001",
  "source": "s2l",
  "transaction_type": "Sale",
  "bhk": "3 BHK",
  "sector": "Sector 54",
  "society": "Ireo Skyon",
  "floor": 12,
  "area_sqft": 2250,
  "price_inr": 22500000,
  "furnishing": "Fully Furnished",
  "status": "Ready to Move",
  "facing": "East",
  "contact": "",
  "notes": "",
  "verified": true,
  "created_at": "2025-01-01",
  "updated_at": "2025-01-01"
}
```

---

### `output/review/`
Records flagged by the extraction pipeline with low confidence or conflicting field values. Each file includes a `_review_reason` field explaining what needs checking:

```json
{
  "_review_reason": "Area value '22sq' could not be parsed. Price missing.",
  ...
}
```

Workflow: manually correct → move corrected JSON to `output/master/`.

---

### `output/logs/`
One log file per extraction run. Filename format: `YYYY-MM-DD_HH-MM_<source>.log`

Log entries track: files processed, records extracted, fields with low confidence, deduplication hits, errors.

---

### `output/exports/`
Ready-to-share files generated from `output/master/`:
- `inventory_<date>.csv` — flat CSV for CRM / spreadsheet use
- `inventory_<date>.xlsx` — formatted Excel workbook
- `inventory_<date>.json` — full JSON array for API / website import

---

### `photos/temp/`
Drop raw property photos here before sorting. Use the organisation script in `scripts/` to group them by listing ID automatically, or move manually.

Naming convention before sorting: `<society>_<bhk>_<floor>_<YYYYMMDD>_<seq>.jpg`

---

### `photos/listings/`
Photos organised by listing ID. One subfolder per listing:

```
listings/
└── MRH-0042/
    ├── 01-living.jpg
    ├── 02-master-bed.jpg
    ├── 03-kitchen.jpg
    └── 04-balcony.jpg
```

---

### `scripts/`
Place OCR, normalisation, and export scripts here.

Suggested scripts (to be created):

| Script | Purpose |
|--------|---------|
| `extract.mjs` | Run OCR on `raw/` screenshots, write to `processed/` |
| `normalise.mjs` | Apply `config/` rules to `processed/`, write to `output/master/` and `output/review/` |
| `export.mjs` | Generate CSV / XLSX from `output/master/` into `output/exports/` |
| `sort-photos.mjs` | Move photos from `photos/temp/` into `photos/listings/<id>/` |

---

### `config/`

| File | Purpose |
|------|---------|
| `normalization-rules.json` | BHK aliases, area unit conversions, price multipliers, furnishing and status aliases |
| `floor-mapping.json` | Floor description strings → canonical integer floor number |
| `source-priority.json` | Source trust ranking for deduplication, per-field overrides |
| `sector-mapping.json` | Raw sector name variants → canonical sector label + area group + PIN |

---

## Workflow

```
1. Drop raw files into raw/<source>/
2. Run: node scripts/extract.mjs
3. Run: node scripts/normalise.mjs
4. Review records in output/review/ → move approved to output/master/
5. Run: node scripts/export.mjs
6. Import output/master/ or output/exports/ into the website
```

---

## Notes

- All paths in this workspace are relative to `inventory/`.
- No file in this folder is imported, bundled, or served by Next.js.
- Add `inventory/raw/`, `inventory/processed/`, `inventory/photos/` to `.gitignore` if this repo is shared — they may contain sensitive pricing or contact data.
